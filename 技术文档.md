## 一、技术选型

### 1.1 整体架构分层

```
┌──────────────────────────────────────────────────────────────┐
│                    展示层 (Presentation)                       │
│  React 18 + TypeScript + TailwindCSS + Zustand               │
│  悬浮窗 · 候选词窗口 · 设置面板 · 引导页 · 托盘菜单            │
├──────────────────────────────────────────────────────────────┤
│                    通信层 (Bridge)                             │
│  Wails Bind (Go ↔ JS 双向 RPC) + Wails Events (服务端推送)    │
├──────────────────────────────────────────────────────────────┤
│                    业务层 (Business)                           │
│  音频采集 · 拼音引擎 · ASR调度 · 文本注入 · 快捷键 · 词库      │
├──────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure)                  │
│  Win32 API · SQLite · Sherpa-ONNX · 云端ASR · 日志 · 守护进程  │
└──────────────────────────────────────────────────────────────┘
```

### 1.2 详细技术选型

#### 1.2.1 应用框架

| 模块 | 选型 | 版本 | 说明 |
|------|------|------|------|
| 桌面框架 | Wails | v2.9+ | Go 后端 + WebView2 前端，单二进制分发 |
| 前端框架 | React | 18.x | 组件化 UI，生态丰富 |
| 类型系统 | TypeScript | 5.x | 类型安全，Wails 自动生成类型定义 |
| CSS 框架 | TailwindCSS | 3.x | 原子化 CSS，快速构建 UI |
| 状态管理 | Zustand | 4.x | 轻量（< 1KB），适合工具类应用 |
| 构建工具 | Vite | 5.x | 快速 HMR，开箱即用 |

**选择理由**：
- Wails 复用系统 WebView2，不内嵌 Chromium，打包体积 ~25MB（vs Electron ~120MB）
- Go 原生 syscall 调用 Win32 API，无需 IPC 桥接或 native addon
- Zustand 比 Redux 轻量一个数量级，适合单窗口工具类应用的简单状态

#### 1.2.2 后端核心模块

| 模块 | 选型 | 说明 |
|------|------|------|
| 后端语言 | Go 1.22+ | 单二进制、goroutine 并发、原生 syscall |
| 音频采集 | PortAudio (`github.com/gordonklaus/portaudio`) | WASAPI 低延迟后端，16kHz/单声道/16bit |
| 在线 ASR | 讯飞实时语音识别 WebSocket API | 中文准确率 ≥ 95%，流式返回，首字 < 200ms |
| 离线 ASR | Sherpa-ONNX (CGo 调用) | 本地推理，< 300ms 延迟，模型 40-80MB |
| 拼音引擎 | Go 自研拼音解析器 | 音节切分 + Trie 查找，全拼/简拼支持 |
| 候选词排序 | N-gram 二元语言模型 | 基于词频+上下文的智能排序 |
| 文本注入 | Win32 SendInput + Clipboard API | Go syscall 直接调用，兼容所有应用 |
| 全局快捷键 | Win32 SetWindowsHookEx | 系统级键盘钩子，焦点无关 |
| 键盘钩子 | Win32 Raw Input / GetAsyncKeyState | 捕获按键实现拼音输入 |
| 候选词窗口定位 | Win32 GetCaretPos + GetGUIThreadInfo | 获取当前光标屏幕坐标 |
| 剪贴板操作 | Win32 OpenClipboard / SetClipboardData | 长文本注入备选方案 |
| 悬浮窗 | Wails Frameless Window | 透明无边框，可拖拽，WebView2 渲染 |

#### 1.2.3 存储方案

| 数据类型 | 存储方式 | 说明 |
|------|------|------|
| 应用配置 | SQLite (`modernc.org/sqlite`) | 纯 Go SQLite 实现，无 CGO 依赖 |
| 用户词库 | SQLite + 内存 Trie 缓存 | 词频表 + 前缀索引，启动时加载到内存 |
| 系统词库 | 嵌入式文件 (embed.FS) | 编译时嵌入二进制，约 30 万词条 |
| 输入历史 | SQLite | 限保留最近 30 天，自动清理 |
| 语音缓存 | 内存 (不落盘) | 识别后即丢弃，保护隐私 |
| 日志文件 | 文件轮转 (`lumberjack`) | 按日切割，保留 7 天 |
| 离线模型 | 本地文件 (`%APPDATA%/guyuInput/models/`) | 首次使用自动下载 |

#### 1.2.4 ASR 引擎对比

| 引擎 | 准确率 | 延迟 | 离线 | 成本 | 中文标点 | 选择 |
|------|--------|------|------|------|----------|------|
| 讯飞语音听写 | ★★★★★ (≥95%) | < 200ms | ❌ | ¥0.0033/次 | ★★★★★ | **首选在线** |
| 阿里云 ASR | ★★★★★ (≥94%) | < 300ms | ❌ | ¥0.0125/min | ★★★★☆ | 备选在线 |
| Sherpa-ONNX (中文) | ★★★☆☆ (≈90%) | < 300ms | ✅ | 免费 | ★★★☆☆ | **离线兜底** |
| Whisper.cpp tiny | ★★★☆☆ (≈88%) | 300-800ms | ✅ | 免费 | ★★☆☆☆ | 不选用（太慢） |

**推荐策略**：讯飞在线优先 → 失败降级 Sherpa-ONNX 离线 → 后台检测恢复后切回在线

```go
type AudioCapture interface {
    // Start 开始录音，返回音频数据通道（float32, 16kHz, 单声道）

    Volume() float32

    // IsRecording 是否正在录音
    IsRecording() bool

    // ListDevices 列出可用音频输入设备
    ListDevices() ([]AudioDevice, error)

    // SelectDevice 选择指定音频设备
    SelectDevice(deviceID string) error
}

type AudioDevice struct {
    ID       string `json:"id"`
    Name     string `json:"name"`
    IsDefault bool  `json:"is_default"`
}
```

#### 2.1.2 ASR 引擎接口

```go
// ASREngine 语音识别引擎接口
type ASREngine interface {
    // Recognize 流式识别，输入音频数据通道，输出识别结果通道
    // resultCh 输出两种结果：
    //   - 前缀为 "partial:" 表示中间结果（边说边出字）
    //   - 前缀为 "final:" 表示最终结果（一句话识别完成）
    Recognize(audioCh <-chan []float32) (<-chan string, error)

    // Mode 返回引擎模式
    Mode() ASRMode

    // Close 释放资源
    Close() error
}

type ASRMode string

const (
    ASRModeOnline  ASRMode = "online"
    ASRModeOffline ASRMode = "offline"
    ASRModeAuto    ASRMode = "auto"
)

// ASRDispatcher ASR 调度器（在线/离线自动切换）
type ASRDispatcher struct {
    online  ASREngine
    offline ASREngine
    mode    ASRMode
}
```

#### 2.1.3 键盘输入引擎接口

```go
// PinyinEngine 拼音输入引擎接口
type PinyinEngine interface {
    // ProcessKey 处理按键事件，返回候选词结果
    ProcessKey(key rune) *CandidateResult

    // Reset 重置输入缓冲区
    Reset()

    // GetPinyinBuf 获取当前拼音缓冲区
    GetPinyinBuf() string

    // LoadUserDict 从数据库加载用户词库到内存
    LoadUserDict() error

    // SaveUserDict 将内存中的词频变更持久化到数据库
    SaveUserDict() error

    // ImportDict 导入外部词库文件
    ImportDict(filePath string, dictType DictType) error
}

type CandidateResult struct {
    Pinyin     string   `json:"pinyin"`      // 当前拼音
    Candidates []string `json:"candidates"`  // 候选词列表（最多5个）
    Committed  string   `json:"committed"`   // 已确认上屏的文字
    IsComplete bool     `json:"is_complete"` // 输入是否完成
}

type DictType string

const (
    DictTypeSystem    DictType = "system"    // 系统词库
    DictTypeUser      DictType = "user"      // 用户词库
    DictTypeCustom    DictType = "custom"    // 专业词库
)
```

### 2.3 Wails Events 推送（Go → 前端）

后端通过 Wails Events 向前端实时推送数据：

```go
// 事件名称常量（Go 端）
const (
    EventAudioVolume    = "audio-volume"     // 音量更新，payload: float32
    EventASRPartial     = "asr-partial"      // 语音识别中间结果，payload: string
    EventASRFinal       = "asr-final"        // 语音识别最终结果，payload: string
    EventASRModeChanged = "asr-mode-changed" // ASR 模式变更，payload: string
    EventInputModeChanged = "input-mode-changed" // 输入模式变更，payload: string
    EventCandidates     = "candidates"       // 候选词更新，payload: CandidateResult
    EventStatusChanged  = "status-changed"   // 状态变更，payload: AppStatus
    EventDictUpdated    = "dict-updated"     // 词库更新，payload: DictStats
    EventConfigChanged  = "config-changed"   // 配置变更，payload: {key, value}
    EventError          = "app-error"        // 错误通知，payload: ErrorInfo
)
```

```typescript
// 前端接收示例（TypeScript）
import { EventsOn } from '../wailsjs/runtime/runtime'

// 音量实时显示
EventsOn('audio-volume', (vol: number) => {
    setVolume(Math.round(vol * 100))
})

// 识别结果流式更新
EventsOn('asr-partial', (text: string) => {
    appendPartialText(text) // 实时刷新光标位置文字
})

EventsOn('asr-final', (text: string) => {
    commitText(text) // 一句话完成，确认上屏
})

// 候选词更新
EventsOn('candidates', (result: CandidateResult) => {
    setCandidates(result.candidates)
    setPinyin(result.pinyin)
})

// 输入模式切换
EventsOn('input-mode-changed', (mode: string) => {
    setInputMode(mode) // 'voice' | 'keyboard' | 'idle'
})
```

### 2.5 数据模型（SQLite）

```sql
-- 用户词库表
CREATE TABLE user_dict (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    word       TEXT    NOT NULL,          -- 词汇
    pinyin     TEXT    NOT NULL,          -- 拼音（无调号）
    frequency  INTEGER DEFAULT 1,        -- 词频（越大越优先）
    dict_type  TEXT    DEFAULT 'user',   -- system/user/custom
    created_at INTEGER DEFAULT (strftime('%s','now')),
    updated_at INTEGER DEFAULT (strftime('%s','now'))
);

CREATE INDEX idx_user_dict_pinyin ON user_dict(pinyin);
CREATE INDEX idx_user_dict_freq ON user_dict(frequency DESC);

-- 应用配置表
CREATE TABLE config (
    key        TEXT PRIMARY KEY,
    value      TEXT    NOT NULL,
    updated_at INTEGER DEFAULT (strftime('%s','now'))
);

-- 输入历史表（本地存储，自动清理）
CREATE TABLE input_history (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    text       TEXT    NOT NULL,          -- 输入内容
    source     TEXT    NOT NULL,          -- voice/keyboard
    app_name   TEXT,                     -- 目标应用名称
    created_at INTEGER DEFAULT (strftime('%s','now'))
);

CREATE INDEX idx_history_date ON input_history(created_at);

-- 输入统计表
CREATE TABLE input_stats (
    date       TEXT    PRIMARY KEY,       -- YYYY-MM-DD
    voice_chars  INTEGER DEFAULT 0,      -- 语音输入字数
    kbd_chars    INTEGER DEFAULT 0,      -- 键盘输入字数
    voice_count  INTEGER DEFAULT 0,      -- 语音使用次数
    kbd_count    INTEGER DEFAULT 0       -- 键盘使用次数
);
```

### 3.1 异常分类

#### 3.1.1 错误码定义

```go
// ErrorCode 统一错误码
type ErrorCode int

const (
    // 通用错误 (1000-1099)
```

### 3.2 重试策略

```go
// RetryPolicy 重试策略
type RetryPolicy struct {
    MaxRetries   int           // 最大重试次数
    BaseDelay    time.Duration // 基础延迟
    MaxDelay     time.Duration // 最大延迟
    Multiplier   float64       // 退避乘数
}

// 各场景重试策略配置
var (
    // 网络请求重试：3次，100ms → 200ms → 400ms
    NetworkRetryPolicy = RetryPolicy{
        MaxRetries: 3,
        BaseDelay:  100 * time.Millisecond,
        MaxDelay:   2 * time.Second,
        Multiplier: 2.0,
    }

    // 剪贴板重试：3次，50ms → 100ms → 200ms
    ClipboardRetryPolicy = RetryPolicy{
        MaxRetries: 3,
        BaseDelay:  50 * time.Millisecond,
        MaxDelay:   500 * time.Millisecond,
        Multiplier: 2.0,
    }

    // 音频设备重试：不重试，立即返回错误（用户需手动干预）
    AudioRetryPolicy = RetryPolicy{
        MaxRetries: 0,
    }
)

// Retry 通用重试执行器
func Retry[T any](policy RetryPolicy, fn func() (T, error)) (T, error) {
    var lastErr error
    delay := policy.BaseDelay

    for i := 0; i <= policy.MaxRetries; i++ {
        if result, err := fn(); err == nil {
            return result, nil
        } else {
            lastErr = err
        }

        if i < policy.MaxRetries {
            time.Sleep(delay)
            delay = time.Duration(float64(delay) * policy.Multiplier)
            if delay > policy.MaxDelay {
                delay = policy.MaxDelay
            }
        }
    }
    var zero T
    return zero, fmt.Errorf("retry exhausted after %d attempts: %w",
        policy.MaxRetries+1, lastErr)
}
```

### 3.3 降级策略

```go
// DegradationStrategy 降级策略
type DegradationStrategy struct {
    mu       sync.Mutex
    status   map[string]DegradeStatus // 各模块降级状态
}

type DegradeStatus int

const (
    DegradeNormal    DegradeStatus = 0 // 正常
    DegradeLimited   DegradeStatus = 1 // 部分功能降级
    DegradeMinimal   DegradeStatus = 2 // 最小功能运行
    DegradeFatal     DegradeStatus = 3 // 不可用
)

// 降级矩阵
//
// 场景                            | 触发条件                | 降级行为
// ─────────────────────────────────────────────────────────────────
// 在线 ASR 不可用                 | 网络断开/API错误         | → 切换离线 ASR（准确率下降≈5%）
// 离线模型加载失败                | 模型文件损坏/磁盘满       | → 仅在线模式（需网络）
// 在线+离线均不可用               | 断网+无模型              | → 仅键盘输入可用
// 剪贴板不可用                    | 被其他应用锁定           | → 切换 SendInput 逐字符注入
// SendInput 被拦截                | 游戏反作弊/安全软件      | → 剪贴板注入兜底
// 用户词库损坏                    | DB文件损坏               | → 重建空词库 + 仅用系统词库
// 全部注入方式失败                | 极端安全策略             | → 复制到剪贴板 + 通知用户手动粘贴
```

```go
func PageOk[T any](items []T, total int64, page, pageSize int) Result[PageResult[T]] {
    totalPages := int(math.Ceil(float64(total) / float64(pageSize)))

        Page:       page,
```

### 4.5 前端类型定义

```typescript
// TypeScript 类型定义（与 Go 结构一一对应）

interface Result<T> {
    code: number
    message: string
    data: T
    trace_id: string
    ts: number
}

interface PageResult<T> {
    items: T[]
    total: number
    page: number
    page_size: number
    total_pages: number
}

interface PageRequest {
    page?: number
    page_size?: number
}

interface Event<T> {
    type: string
    data: T
    ts: number
    trace_id: string
}

// 前端 Result 工具函数
function isOk<T>(result: Result<T>): boolean {
    return result.code === 0
}

function unwrap<T>(result: Result<T>): T {
    if (!isOk(result)) {
        throw new AppError(result.code, result.message, result.trace_id)
    }
    return result.data
}

class AppError extends Error {
    constructor(
        public code: number,
        message: string,
        public traceId: string
    ) {
        super(message)
        this.name = 'AppError'
    }
}
```

### 4.6 TraceID 生成规则

```go
// TraceID 格式：{timestamp}-{random}
// 示例: 1a2b3c4d-5e6f
func newTraceID() string {
    b := make([]byte, 4)
    rand.Read(b)
    return fmt.Sprintf("%x-%x", time.Now().UnixNano()/1e6, b)
}
```

- 每个用户操作（录音开始、选词、配置变更）都生成唯一的 TraceID
- TraceID 贯穿 Go 后端 → Wails Bridge → React 前端 → 日志文件，实现全链路追踪
- 用户反馈问题时，只需提供 TraceID 即可在日志中定位
